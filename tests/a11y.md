# Checklist de Acessibilidade (WCAG 2.1 AA)

Este documento serve como guia para validar a conformidade com a **WCAG 2.1 Nível AA** no jogo **Desafio da Prevenção de Perdas**. A acessibilidade deve ser considerada desde o início do desenvolvimento.

## Textos e Cores

- [ ] **Contraste**: Verifique se texto e ícones possuem contraste mínimo de 4.5:1 (3:1 para títulos grandes) em todos os modos de cores.
- [ ] **Tamanho de Fonte**: Garantir tamanho mínimo de 14 px e permitir zoom sem quebrar layout.
- [ ] **Texto Alternativo**: Imagens (ex.: logotipo) possuem atributos `alt` descritivos.

## Navegação

- [ ] **Teclado**: Todos os controles (botões, inputs, links) são acessíveis via teclado (`Tab`, `Enter`, `Espaço`).
- [ ] **Foco Visível**: O foco do teclado tem indicação visual clara e não é removido via CSS.
- [ ] **Ordem Lógica**: A ordem de tabulação segue a estrutura visual e semântica da página.

## Semântica e ARIA

- [ ] **Elementos Semânticos**: Uso de `<header>`, `<main>`, `<section>`, `<footer>`, `<nav>` (quando aplicável).
- [ ] **Rótulos**: Inputs possuem `<label>` associado via `for` e `id`.
- [ ] **ARIA**: Atributos `aria-live`, `role` e outros são utilizados para anunciar mudanças dinâmicas (cronômetro, explicações).
- [ ] **Language**: O atributo `lang="pt-BR"` está presente no elemento `<html>`.

## Conteúdo Dinâmico

- [ ] **Anúncio de Mudanças**: Perguntas e timers atualizam com `aria-live` apropriado para leitores de tela.
- [ ] **Tempo Suficiente**: Jogadores têm opção de pausar (não aplicável neste jogo, mas tempo razoável de 15 s é oferecido).

## Mídia e Som

- [ ] **Autoplay**: Sons não iniciam sem interação do usuário; são disparados apenas após ação do jogador.
- [ ] **Alternativas de Som**: Caso áudio não funcione, há fallback visual ou vibratório (explicação na tela) e WebAudio.

## Responsividade

- [ ] **Layout Flexível**: O design adapta-se a diferentes tamanhos de tela sem perda de funcionalidade.
- [ ] **Elementos Touch**: Botões e áreas clicáveis possuem tamanho mínimo de 44×44 px para interação confortável em touch screens.

Marque cada item após verificação e corrija conforme necessário para garantir acessibilidade para todos os jogadores.
